03 — AI Workflow
How I work
with AI.

AI isn't a shortcut — it's a multiplier. Here's how I actually use AI tools in my development and learning workflow. Every step involves a decision; the output reflects the quality of those decisions.

01

Research & direction setting

Before writing any code or content, I use AI to research the professional landscape — real job postings, skill requirements, what practitioners in a field actually do. This shapes the strategy, not just the words.

→ Tools used: Claude, Perplexity
02

Ideation & project framing

I use AI to brainstorm project concepts that are realistic for my stage but ambitious in direction. I iterate on ideas, push back on vague concepts, and ask the model to poke holes in my reasoning.

→ Tools used: Claude, ChatGPT
03

Writing & tone refinement

I draft first, then use AI to refine — not to replace. The voice has to sound like me. I compare AI suggestions against my drafts and pick what's true, not just what's polished. Everything you read here started as my words.

→ Tools used: Claude, Grammarly
04

Code generation & debugging

AI accelerates my frontend work significantly. I use it to generate boilerplate, explore layout options, and debug tricky CSS or logic issues. But I read every line before merging it. The goal is to understand, not just to ship.

→ Tools used: Claude, Cursor, GitHub Copilot
05

Image & visual generation

Visual identity matters. I use generative image tools to create consistent, intentional imagery for projects — experimenting with prompts to get visuals that match the tone of the product, not just generic stock.

→ Tools used: Midjourney, DALL-E, Adobe Firefly
